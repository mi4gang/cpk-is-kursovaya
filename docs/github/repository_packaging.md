# Стандарт оформления GitHub-репозитория для защиты

## Позиционирование
Репозиторий должен подтверждать, что проект:
- воспроизводим;
- структурирован;
- привязан к критериям курсовой;
- готов к демонстрации в ограниченное время.

## Обязательные элементы
- `README.md` как единая входная точка.
- `docs/spec` с формализацией требований и критериев.
- `docs/diagrams` с полным набором схем.
- `docs/pz` и `docs/presentation` для защиты.
- `docs/screenshots` для визуальной верификации.
- `.github/workflows/ci.yml` для базовой технической дисциплины.

## Формула демонстрации комиссии
1. Показать README.
2. Показать онлайн-ссылку/локальный запуск.
3. Показать ключевой процесс в системе.
4. Показать диаграммы и ПЗ в `docs/`.
5. Показать релизный тег `v1.0-kursovaya`.
