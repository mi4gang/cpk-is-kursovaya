# Smoke-checklist (ручная приемка MVP)

## Подготовка
1. Запустить приложение: `./scripts/start-local.sh`.
2. Открыть: `http://localhost:8080/` (публичная главная).
3. Проверить CTA `Вход` и `Регистрация`.
4. Проверить `http://localhost:8080/login`.
5. Без логина открыть `http://localhost:8080/programs`:
- каталог открывается;
- карточки программ доступны;
- кнопка консультации ведет на `/consultation`.
6. Демо-аккаунты:
- `admin / admin123`
- `methodist / method123`
- `teacher / teacher123`
- `student / student123`

## Базовая безопасность
1. Без логина открыть `/admin/dashboard-v2` -> ожидается редирект на `/login`.
2. Войти под каждой ролью и открыть `/dashboard`:
- `admin` -> редирект на `/admin/dashboard-v2`;
- `methodist` -> редирект на `/methodist/queue`;
- `teacher` -> редирект на `/teacher/groups`;
- `student` -> редирект на `/student/cabinet`.
3. Под `student` открыть `/admin/dashboard-v2` -> ожидается отказ доступа.
4. Под `teacher` открыть `/payments` -> ожидается отказ доступа.
5. Под `methodist` открыть `/assessments` -> ожидается отказ доступа.

## Роль STUDENT
1. Войти под `student`.
2. Открыть `/student/cabinet`:
- видна карточка этапа;
- виден прогресс;
- виден блок следующего шага;
- видны рекомендации по категории.
3. Создать заявку через `/applications/new`.
4. Проверить, что в `/applications` студент видит только свои заявки.

## Роль METHODIST
1. Войти под `methodist`.
2. Открыть `/methodist/queue`.
3. В очереди документов:
- подтвердить документы одной заявки;
- отклонить документы другой заявки.
4. В очереди доступа:
- открыть trial на 3 дня;
- назначить преподавателя.
5. Открыть `/payments` и провести оплату (статус `PAID`).

## Роль TEACHER
1. Войти под `teacher`.
2. Открыть `/teacher/groups`.
3. Обновить прогресс у назначенного слушателя.
4. Открыть `/assessments`, внести результат:
- балл `>= 75` -> статус `Пройдена`;
- балл `< 75` -> статус `Не пройдена`.
5. После `PASSED` нажать «Завершить» в `/teacher/groups`.

## Роль METHODIST (выдача удостоверения)
1. Вернуться под `methodist`.
2. Открыть `/methodist/queue` и убедиться, что заявка в очереди выдачи удостоверений.
3. Открыть `/certificates/new` и оформить удостоверение.
4. Указать `documentLink` (ссылка на документ).

## Финальная проверка e2e
1. Войти под `student`.
2. Открыть `/student/cabinet`.
3. Проверить:
- завершение обучения зафиксировано;
- удостоверение отображается;
- ссылка на документ доступна.

## Админский контроль
1. Войти под `admin`.
2. Открыть `/admin/dashboard-v2`.
3. Проверить KPI:
- всего слушателей;
- активные программы;
- сумма оплат;
- очередь документов;
- очередь доступа;
- очередь удостоверений;
- нагрузка преподавателей;
- спрос по программам (BI-намек).
