# Спецификация требований (1 страница)

## Проект
Информационная система центра повышения квалификации (ИС ЦПК).

## Цель
Обеспечить учет программ обучения, заявок слушателей, оплат, результатов аттестации и выдачи удостоверений в едином интерфейсе с разграничением ролей.

## Линейный бизнес-процесс
Программа -> Заявка слушателя -> Оплата -> Результат аттестации -> Удостоверение.

## Роли и права
- Администратор: полный доступ.
- Методист: ведение программ, заявок, оплат и удостоверений.
- Преподаватель: ведение результатов аттестации, просмотр связанных данных.
- Слушатель: создание заявки и просмотр данных в рамках общего интерфейса.

## Обязательный функционал MVP
- Авторизация и разграничение доступа по ролям.
- CRUD для сущностей: Program, Application, Payment, AssessmentResult, Certificate.
- Поиск: по названию программы и ФИО слушателя.
- Сортировка: по основным полям (дата, название, статус, сумма).
- Статистика на главной странице: общее число слушателей, активные программы, сумма оплаченных платежей.
- Страница «Об авторе» с формальными полями по методичке.
- Единая обработка ошибок (404, 500, ошибки валидации) с понятным сообщением.

## Нефункциональные требования
- Архитектура: трехзвенная (UI -> сервер -> БД).
- Стек: Java, Spring Boot, Spring Security, Spring Data JPA, PostgreSQL, Thymeleaf.
- Устойчивость: исключение аварийного завершения, пользовательские сообщения об ошибках.
- Безопасность: ORM и шаблонизатор для снижения рисков SQL-инъекций и XSS.
- Простота сопровождения: модульная структура контроллеров и репозиториев.

## Ограничения
- Реализация в учебном формате без сложных внешних интеграций.
- Фокус на формальном соответствии критериям курсовой работы.
